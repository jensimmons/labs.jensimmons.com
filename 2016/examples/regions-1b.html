Responsive text, a la 2011

For a while there, web designers make layouts with fixed-width containers, allowing us to create good lines lengths for text, and to control the graphic design of page layout better. But those ideas were rooted in a consensual hallucination that all the screens people used to get to the internet were the same size. The explosion of tiny phone screens blew up that idea. Responsive design emerged, and we started creating fluid designs again, but this time setting maximum-widths on text containers so the line length of our words was nice and readable.

The problem is that this design is boring. Everything starts to look the same. We can change the typeface, place other items on the page â€” like big photographs. We don't have to put the single column of text centered on the page; we can use uneven margins. But we are still stuck with a layout that's basically one long narrow column of text running in a straight line down the page. We can't stagger that column or break things into two columns. And that is incredibly limiting. Magazines and newspapers have a long tradition of using multiple columns. So let's see in the next example what's possible on the web with multicolumn.
